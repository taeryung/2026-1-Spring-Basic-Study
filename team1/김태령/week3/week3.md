#Spring-Basic-Week3

## 백엔드와 프론트엔드
* 웹 개발 초기와 비교했을 때, 현대 웹 개발에서 프론트엔드와 백엔드 역할이 분리된 주된 이유는 **웹 기능이 복잡**해지고 **사용자 상호작용**이 증가하면서 **화면과 데이터 처리를 분리**하는 것이 효율적이었기 때문
* 화면 개발을 담당하는 것이 **프론트엔드**, 로직 개발 및 데이터 처리를 담당하는 것이 **백엔드**

## API
* 백엔드 개발자는 API를 만듦
* Application Programming Interface
* 백엔드 개발자가 만드는 핵심적인 것 중 하나
* 다른 시스템이나 프론트엔드가 **백엔드의 기능이나 데이터에 접근**할 수 있게 해주는 인터페이스 역할을 함
* 애플리케이션이 서로 상호작용하기 위해 사용하는 인터페이스
* REST API : 인터넷 세상에서 지켜야 하는 약속인 HTTP 원칙을 잘 지키는 API

## API를 사용하는 이유
* 시스템이 자신의 데이터를 외부 애플리케이션이나 사용자에게 제공할 때, **데이터베이스에 대한 직접 접근 대신 API**를 사용
* 개개인이 데이터에 마음대로 접근하게 하기에는 위험성이 존재
* **내부 시스템 구조를 숨기고, 데이터 접근을 통제**해 보안과 안정성을 확보하기 위함
* interface를 세우는 것은 데이터 앞에 가림막을 세우는 것과 동일

## Spring Initializer
* 스프링 부트 프로젝트의 기반을 생성. 초기 뼈대와 설정을 자동으로 생성 (https://start.spring.io/)
* **project: gradle - Groovy**와 같은 빌드 도구는 소프트웨어 개발에서 소스를 실행 가능한 형태로 만드는 과정을 자동화
* **Language: Java**
* **Project Metadata**는 프로젝트를 밖에서 봤을 때 설정 
* **Jar : Java archive / War: Web archive** : 스프링 부트 웹 애플리케이션 배포 시, JAR 방식이 선호됨. 내장 웹 서버까지 포함해서 단일 실행 파일로 만들어주기 때문
* **Dependencies: springboot web**선택하는 것은 다른 라이브러리나 프레임워크 기능을 사용하는 관계를 의미.
